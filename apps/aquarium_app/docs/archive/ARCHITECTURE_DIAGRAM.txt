┌─────────────────────────────────────────────────────────────────────────────┐
│                           DATA FLOW                                         │
└─────────────────────────────────────────────────────────────────────────────┘

1. User taps "Take Quiz" in Lesson
        ↓
2. Navigate to EnhancedQuizScreen(quiz)
        ↓
3. Screen renders first exercise via ExerciseWidget factory
        ↓
4. User interacts with exercise-specific widget
        ↓
5. Widget calls onAnswer(userAnswer) callback
        ↓
6. Screen stores answer in Map<int, dynamic>
        ↓
7. User taps "Check Answer"
        ↓
8. Screen calls exercise.validate(answer)
        ↓
9. Show feedback (green=correct, red=incorrect)
        ↓
10. Display explanation card
        ↓
11. User taps "Next Question"
        ↓
12. Repeat 3-11 for each exercise
        ↓
13. Show results screen with score, percentage, XP
        ↓
14. User taps "Complete"
        ↓
15. Call onQuizComplete(score, maxScore, bonusXp)
        ↓
16. Return to previous screen

┌─────────────────────────────────────────────────────────────────────────────┐
│                        VALIDATION LOGIC                                     │
└─────────────────────────────────────────────────────────────────────────────┘

MultipleChoice:  answer (int) == correctIndex
FillBlank:       answer (List<String>) matches correctAnswers[]
                 (with case-insensitive + alternatives support)
TrueFalse:       answer (bool) == correctAnswer
Matching:        answer (Map<int,int>) == correctPairs
Ordering:        answer (List<int>) == correctOrder

┌─────────────────────────────────────────────────────────────────────────────┐
│                     DIFFICULTY CALCULATION                                  │
└─────────────────────────────────────────────────────────────────────────────┘

Easy:    • True/False (always)
         • Fill-blank with word bank
         • Multiple choice (2 options)
         • Matching (≤3 pairs)
         • Ordering (≤3 items)

Medium:  • Multiple choice (3-4 options)
         • Fill-blank (1 blank, no word bank)
         • Matching (4-5 pairs)
         • Ordering (4-5 items)

Hard:    • Multiple choice (5+ options)
         • Fill-blank (2+ blanks)
         • Matching (6+ pairs)
         • Ordering (6+ items)

┌─────────────────────────────────────────────────────────────────────────────┐
│                      ANIMATION TIMELINE                                     │
└─────────────────────────────────────────────────────────────────────────────┘

Option Selection:
    0ms:  User taps → Scale down to 0.98
  100ms:  Scale back to 1.0
  300ms:  Border color changes (blue)
          Background color fades in

Answer Check:
    0ms:  User taps "Check" → Scale animation
  200ms:  Correct options turn green
          Incorrect options turn red
  400ms:  Explanation card slides in
          Icons appear (✓ or ✗)

Next Question:
    0ms:  User taps "Next"
  300ms:  Current exercise fades out
  600ms:  Progress bar animates
  900ms:  New exercise fades in

Results Screen:
    0ms:  Quiz completes
  300ms:  Emoji icon scales up (elastic)
  800ms:  Score text appears
 1200ms:  Percentage circle animates
 1800ms:  XP card slides up
 2200ms:  Button appears
