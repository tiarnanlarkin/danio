## 4. BROKEN / UNREACHABLE ACHIEVEMENTS

### 🔴 CONFIRMED BROKEN (will never fire under any normal circumstances)

#### `speed_demon` — Speed Demon
- **Problem:** `lessonDuration` is hardcoded to `widget.lesson.estimatedMinutes * 60` (estimated reading time), NOT actual elapsed time. The minimum lesson estimate in the app is 4 minutes (240 seconds). The `speed_demon` threshold is 120 seconds. Since `240 >= 120` is always false, this achievement **can never unlock**.
- **Fix needed:** Track actual lesson start time and pass real elapsed seconds to `checkAfterLesson`.

#### `comeback` — The Comeback  
- **Problem:** The `comeback` check requires `stats.lastActivityDate != null`, but `lastActivityDate` is **never passed** in any `checkAfterLesson()` call site (or any other). It always defaults to `null`. The guard `if (stats.lastActivityDate != null)` means the unlock logic is never reached.
- **Fix needed:** Pass `userProfile.lastActivityDate` when constructing `AchievementStats` in `checkAfterLesson`.

#### `perfectionist` — Perfectionist (10 perfect quizzes)
- **Problem:** `perfectScores` is calculated as:
  ```dart
  profile.achievements.where((a) => a == 'perfectionist').length + 1
  ```
  This checks if the string `'perfectionist'` appears in the achievements list (which stores achievement IDs). It returns either 0 (not unlocked yet) or 1 (already unlocked). The value can never reach 10, so the achievement **can never unlock**.
- **Fix needed:** Add a dedicated `perfectScoresCount` field to `UserProfile` and increment it on each perfect quiz, then pass it to `AchievementStats`.

#### `social_butterfly` — Social Butterfly (10 friends)
- **Problem:** `checkAfterFriendAdded()` is **never called from any screen**. No UI code triggers friend-based achievement checks.
- **Fix needed:** Call `achievementChecker.checkAfterFriendAdded(friendsCount: newCount)` when the friends list changes.

### 🟡 POTENTIALLY UNREACHABLE (depends on content state)

#### `teachers_pet` — Teacher's Pet
- Requires completing **every lesson** in the app. Achievable in theory but the `allPathMetadata` must have all paths loaded. Currently relies on `LessonProvider.allPathMetadata` at check time being complete — verify no paths are lazy-loaded/missing at check time.

#### `completionist` — Completionist (hidden)
- Checks `userProfile.achievements.length >= 48` (all non-hidden, non-completionist achievements).
- Since `social_butterfly`, `speed_demon`, `comeback`, and `perfectionist` are all broken (see above), users **cannot collect all 48 visible achievements**, making `completionist` also effectively unreachable.
- **Fix:** Fix the 4 broken achievements above first.
