# PRD — Aquarium Hobby App ("Personal Aquarium OS")

**Location:** `Aquarium App Dev/Planning/`
**Status:** Draft v0.1 (planning)
**Product working name:** Aquarium Hobby App (final name TBD)
**Positioning:** A personal aquarium operating system for hobbyists—freshwater first, marine later.

---

## 1) Problem
Aquarium hobbyists end up running their tanks with:
- fragmented tools (notes, spreadsheets, reminders)
- scattered advice (forums, bookmarks, YouTube)
- weak mental models (flat lists that don’t represent "the tank" as a system)

This causes:
- missed maintenance, inconsistent logs
- overstocking/parameter mismatch mistakes
- anxiety and decision fatigue

---

## 2) Target users
### Primary
- Adult beginners → intermediate hobbyists
- Want structure without spreadsheets
- Want guidance without forum hunting
- Enjoy playful UI, but require correctness

### Secondary (later)
- Advanced planted tank keepers (CO₂, high-tech)
- Reef keepers (marine)

---

## 3) Product promise (one-liner)
**“Everything about your aquarium—tanks, livestock, equipment, maintenance, and insights—in one calm, spatial interface.”**

---

## 4) Goals / Success criteria
### Goals
1) Replace spreadsheets/notes for aquarium operations.
2) Make maintenance habitual (tasks feel easy + satisfying).
3) Provide trustworthy guidance that never overstates certainty.

### MVP success criteria
- A beginner can set up a tank in the app and maintain it for 30 days without needing external organisation tools.
- An intermediate user can log water tests + see trends and use reminders.
- Equipment maintenance tracking is genuinely useful (not a token list).

---

## 5) Non-goals (MVP)
- Full social feed/community
- In-app checkout (shopping is links-out only)
- Reef/marine complexity (Phase 7)
- Medical “diagnosis” claims (AI gives observations + next steps, not certainty)

---

## 6) Key differentiators
1) **Spatial metaphor:** rooms + objects reduce cognitive load.
2) **Equipment as first-class objects:** each filter/heater/light has settings + maintenance + logs.
3) **AI with hard accuracy gates:** confidence scoring + “unknown” is allowed.
4) **Calm UX:** no alarm-fatigue, no spammy notifications.

---

## 7) Core objects (data model)
### Tank (first-class object)
- id, name
- type: freshwater (MVP) / marine (later)
- volume (L), dimensions
- start date
- target ranges: temp, pH, GH/KH, NO2/NO3 etc.
- equipment[]
- livestock[]
- logs[]
- tasks[]

### Livestock
- species: common + scientific + canonical ID
- count, size estimate
- date added, source
- temperament flags
- compatibility constraints

### Equipment (first-class)
- type: filter/heater/light/air/feeder/CO₂/etc.
- model/name
- settings (type-specific)
- maintenance schedule + last serviced
- logs + notes

### Logs
- water tests (NH3/NH4, NO2, NO3, pH, GH, KH, temp)
- events (water change %, new fish, meds/treatment, algae outbreak, etc.)
- photos

### Tasks
- generated + user-created
- recurring rules + snooze
- completion history

---

## 8) MVP feature scope
### Must-have (MVP)
1) Account + multi-tank support
2) Create tank wizard (freshwater)
3) Tank dashboard (status + last actions)
4) Logging:
   - water tests
   - temperature
   - events
   - photos
5) Tasks/reminders:
   - autogenerated starter schedule (weekly water change, monthly filter check)
   - snooze/disable
6) Livestock:
   - add via text search
   - species detail page (basic care sheet)
   - basic compatibility checks vs tank targets
7) Equipment:
   - add equipment items
   - per-equipment maintenance schedule + log
8) Basic charts (simple trend lines, optional)

### Should-have (MVP+)
- Simple “state badges” (e.g., “test overdue”, “filter clean due”)
- Export (CSV) for water tests
- Shop Street (links-out) with disclosure tags

### Later (post-MVP)
- 3D/Unity rooms fully interactive
- AI identification (plants first)
- AI explanations/insights
- Predictive coaching
- Marine/reef

---

## 9) UX / UI spec
### Information architecture (concept)
- **Display Room (Home):** shelves of tanks → quick status badges
- Tap a tank → **Tank Room:** tank-centric management
- Tap objects → **Panels** (Setup / Care / Log)
- **Cupboard:** consumables/inventory
- **Notebook/Lab:** logs + charts
- **Shop Street:** shops + links-out + tracking

### MVP UI approach (if 3D isn’t ready)
- Start with a 2D “room-like” layout (cards arranged like shelves) to validate workflow.
- Add 3D later without changing core data contracts.

---

## 10) AI strategy (MVP planning)
AI is not required for MVP, but the system should be built to support it.

### Principles
- No free-text AI that directly changes data.
- Structured outputs only (schema-validated).
- Hard confidence gates; “unknown” is always allowed.

### AI use-cases (later)
- Identification (plants first)
- Compatibility validation
- Trend detection (“signals”, not alarms)
- Explain-what-you-see (grounded in user logs)

---

## 11) Privacy & safety posture
- Default: local-first feel; cloud optional depending on architecture.
- Clear permission rationale for photos/location.
- Avoid medical claims. Guidance is educational, with uncertainty.

---

## 12) Monetisation (optional, later)
- Freemium: limited tanks/history; Pro unlocks unlimited + insights
- Cosmetic packs (room skins)
- Affiliate links in Shop Street (with disclosures)

---

## 13) Definition of done (MVP)
- Create tank → log tests/events → reminders fire correctly → equipment schedules trackable.
- Zero confusing dead-ends.
- Data export works.
- App is calm, stable, and fast.
